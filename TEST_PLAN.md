# Camera Bridge 回归测试流程

## 测试记录

每项记录：测试编号、通过/失败、实际结果、截图或录屏、诊断编号。发生失败时先在“设置 → 诊断与日志”导出诊断包，再重试，避免覆盖现场。

## A. 公司无相机 / 远程测试

### A0 构建

在 PowerShell 7 中执行：

```powershell
$env:JAVA_HOME='D:\code\androidStudio\jbr'
.\gradlew.bat :App:testDebugUnitTest :App:assembleDebug --no-daemon
```

预期：显示 `BUILD SUCCESSFUL`，APK 位于 `App\build\outputs\apk\debug\App-debug.apk`。

### A1 安装与启动

```powershell
$adb="$env:LOCALAPPDATA\Android\Sdk\platform-tools\adb.exe"
& $adb install -r .\App\build\outputs\apk\debug\App-debug.apk
& $adb logcat -c
& $adb shell am force-stop com.yaoyihan.nikonconnect
& $adb shell monkey -p com.yaoyihan.nikonconnect -c android.intent.category.LAUNCHER 1
```

预期：应用正常启动；未连接相机时底部导航仍存在；“下载、LUT、设置”可以直接进入；点击“相册”会提示先连接相机。

### A2 本地图片流程

在下载页选择一张已有图片，依次验证：

1. 查看、双指缩放、旋转。
2. LUT 强度 0%、50%、100%，画面方向与比例不变化。
3. LUT 后再加水印，不能恢复成未调色原图。
4. 导出 LUT、导出 LUT + 水印，均生成新文件且不覆盖原文件。
5. 关闭预览后重新打开导出文件，方向、色彩、水印和 EXIF 信息正确。

### A3 本地视频流程

使用已有小视频验证：

1. 不套 LUT 时立即有画面和声音。
2. 全屏、暂停、拖动进度、顺时针旋转正常。
3. 套 LUT 后预览生效；调节强度后画面连续变化。
4. 开始导出后切后台 30 秒或锁屏，再返回；进度不能倒退或重新开始。
5. 输出视频能播放和拖动，分辨率、方向、时长正确，音轨存在。

### A4 大视频与空间提示

如果下载页已经有超过 1 GB 的视频：

```powershell
& $adb shell df -h /data /storage/emulated/0
```

套 LUT 导出并记录开始时间、结束时间、导出前后可用空间。预期：不出现 `OutOfMemoryError`；空间不足时在开始生成或保存相册前给出明确的“需要空间 / 当前可用空间”提示。没有大视频时不要修改应用偏好或伪造下载记录，此项留到 B7。

### A5 下载任务 UI

有活动任务时检查：

- 卡片不越界、不叠字；圆环中央只显示等待、百分比、取消中或失败状态。
- 底部显示文件名、已传/总大小、剩余大小和预计时间。
- 总大小未知时显示不定进度及“总大小未知”，已传大小仍持续增长。
- 右上角取消/重试按钮容易点击；取消确认明确说明不会断开相机。
- 分别使用系统字体 100% 与 130% 检查一次，完成后恢复自己的字体设置。

### A6 日志检查

```powershell
& $adb logcat -d -v threadtime |
  Select-String 'FATAL EXCEPTION|AndroidRuntime|OutOfMemoryError|ENOSPC|VIDEO_EXPORT_FAILED|MTP_|ASSET_LIST_FAILED'
```

预期：正常流程没有 `FATAL EXCEPTION`、`OutOfMemoryError` 或未处理的 `ENOSPC`。

## B. 回家连接 Nikon Zf 真机测试

### B1 USB 冷启动与重连

1. 拔掉相机，划掉 Camera Bridge 最近任务。
2. 相机开机并设为 MTP/PTP，重新插线。
3. 选择或允许 Camera Bridge 打开，授权 USB。
4. 预期自动识别并读取相册，“进入相册”始终可点击，即使空卡或读取结果暂时为 0。
5. 在应用前台重复拔插一次，预期复用当前页面并重新检测。

说明：在系统设置中点“强行停止”后，Android 会阻止 USB 广播自动拉起；此时需手动打开应用，不判定为缺陷。

### B2 USB 异常分支

分别测试：首次授权、拒绝后重试、已授权重插、相机 USB 模式错误、空卡。预期都有明确状态和可执行的重试入口，不出现黑屏或永久忙碌。

### B3 分页与视频筛选

相机卡内准备超过 45 个文件，并保证至少一个视频位于第 31 项以后：

1. “全部”滚动到底，确认继续分页。
2. 重新进入相册后直接选择“视频”。
3. 预期会继续读取后续分页并找到视频，不会因首批无视频而显示最终空态。
4. 点击相机视频只出现“下载后播放和编辑”，不能把整个视频当图片加载。

### B4 下载任务与重复点击

分别下载 JPG、RAW、小视频和超过 1 GB 的视频。每次点击后应提示已创建任务；重复点击同一个活动任务只保留一个任务并提示已存在。下载页进度、大小、剩余量和预计时间持续更新。

### B5 队列取消

1. 同时加入三个文件。
2. 取消第二个等待任务：它必须直接消失，之后不能偷偷生成完成文件。
3. 取消第一个正在传输的任务：相机连接保持，第三个任务继续执行。
4. 制造一个失败任务后点击重试；只允许连接创建任务时的原相机执行。

### B6 拔线、后台与进程结束

在传输中分别执行拔线、锁屏、切后台、划掉最近任务。预期：界面不因 MTP 关闭等待而卡死；重连后不会下载错文件；中断任务有明确提示；系统媒体库不留下可见的 0 KB/半文件，应用下次启动会清理自己的待发布文件。

### B7 超过 1 GB 视频 LUT 导出

1. 完整下载超过 1 GB 的 Zf 视频。
2. 记录导出前可用空间，套明显 LUT 并导出。
3. 导出期间观察内存不应随视频时长持续上涨直至崩溃。
4. 核对输出色彩、方向、分辨率、时长、声音和文件大小。
5. 再在空间不足条件下尝试，确认给出精确空间提示且不留下半文件。

### B8 Wi-Fi 对照

用 Wi-Fi 重复 B4、B5 的小文件流程。重点确认取消下载不会关闭 Wi-Fi、后续任务仍能继续、下载页离线仍可查看已完成文件。
